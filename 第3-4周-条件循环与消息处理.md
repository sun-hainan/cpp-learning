## 2.1 if / else：程序的岔路口

**从游戏场景开始：**

你写 Lua 脚本时肯定写过：
```lua
if playerLevel >= 10 then
    enableTransfer(playerId)
else
    sendMessage(playerId, "等级不足")
end
```

C++ 的写法几乎一样：
```cpp
if (playerLevel >= 10) {
    enableTransfer(playerId);
} else {
    sendMessage(playerId, "等级不足");
}
```

**两个区别：**
1. C++ 条件要加括号 `( )`
2. C++ 代码块用 `{ }` 包裹，Lua 用 `then ... end`

**等价的 Lua 对照：**
```lua
-- Lua
if 条件 then
    -- 代码
elseif 条件2 then
    -- 代码
else
    -- 代码
end

-- C++
if (条件) {
    // 代码
} else if (条件2) {
    // 代码
} else {
    // 代码
}
```

## 2.2 if 的坑：赋值 vs 比较

**Lua 的坑（很少见）：**
```lua
if x = 10 then  -- = 是赋值，永远为 true
    print("x是10")
end
```

**C++ 的坑（很常见）：**
```cpp
if (x = 10) {   // 危险！= 是赋值，条件永远为 true！
    cout << "x是10";
}

if (x == 10) {  // 正确：== 才是比较
    cout << "x是10";
}
```

**记忆口诀：**
```
=  是"赋值"（把右边的值放进左边的盒子）
== 是"比较"（问左右两边相等吗）
```

---

## 2.3 for 循环：知道次数的重复

**Lua：**
```lua
for i = 1, 10, 1 do
    print(i)
end
```

**C++：**
```cpp
for (int i = 1; i <= 10; i++) {
    cout << i << " ";
}
```

**逐段解释：**
```cpp
for (
    int i = 1;     // 第1段：初始化（只执行一次）
    i <= 10;        // 第2段：条件判断（每次循环前检查）
    i++             // 第3段：步进（每次循环后执行）
) {
    // 循环体
}
```

**i++ 是什么意思？**
```
i++  等价于  i = i + 1  等价于  i += 1
```

## 2.4 while 循环：不知道次数的重复

**场景：** 服务器循环等待客户端消息，次数不固定

```cpp
// Lua
while true do
    local cmd = waitCommand()
    if cmd == "quit" then break end
end

// C++
while (true) {
    string cmd = waitCommand();
    if (cmd == "quit") break;
}
```

---

## 2.5 for 和 while 的选择

| 场景 | 用什么 |
|------|--------|
| 知道循环几次 | `for` |
| 不知道几次，只知道退出条件 | `while` |
| 至少执行一次 | `do...while` |

## 【实战2A】改进服务器：循环接收多条消息

**上周的服务器只能收一条消息就退出了。这周改成循环接收，直到客户端断开。**

```cpp
// server_v2.cpp
#include <iostream>
#include <winsock2.h>
#include <string>
#pragma comment(lib, "ws2_32.lib")
using namespace std;

int main() {
    // ========== 初始化（省略，同上）==========
    WSADATA wsa;
    WSAStartup(MAKEWORD(2, 2), &wsa);

    SOCKET listenSocket = socket(AF_INET, SOCK_STREAM, 0);
    sockaddr_in serverAddr;
    serverAddr.sin_family = AF_INET;
    serverAddr.sin_port = htons(8888);
    serverAddr.sin_addr.s_addr = INADDR_ANY;

    bind(listenSocket, (sockaddr*)&serverAddr, sizeof(serverAddr));
    listen(listenSocket, 5);
    cout << "[服务器] 监听端口 8888..." << endl;

    // ========== 接受连接 ==========
    sockaddr_in clientAddr;
    int clientLen = sizeof(clientAddr);
    SOCKET clientSocket = accept(listenSocket, (sockaddr*)&clientAddr, &clientLen);
    cout << "[服务器] 客户端已连接" << endl;

    // ========== 循环接收消息（核心改动）==========
    while (true) {
        char buffer[1024] = {0};
        int recvLen = recv(clientSocket, buffer, sizeof(buffer), 0);

        if (recvLen <= 0) {
            // recvLen <= 0 表示客户端断开连接或出错
            cout << "[服务器] 客户端已断开连接" << endl;
            break;
        }

        buffer[recvLen] = '\0';
        cout << "[收到消息] " << buffer << endl;

        // 简单处理：收到 "quit" 就断开
        if (string(buffer) == "quit") {
            const char* reply = "再见！";
            send(clientSocket, reply, strlen(reply), 0);
            break;
        }

        // 回复客户端
        string reply = "服务器收到: " + string(buffer);
        send(clientSocket, reply.c_str(), reply.length(), 0);
    }

    // ========== 清理 ==========
    closesocket(clientSocket);
    closesocket(listenSocket);
    WSACleanup();
    return 0;
}
```

**关键点：**
- `recv()` 返回值 `> 0` 表示收到数据，`= 0` 表示客户端关闭了连接，`< 0` 表示出错
- `string(buffer)` 把 C 风格字符串转成 C++ string，方便处理
- `.c_str()` 把 C++ string 转回 C 风格字符串，因为 `send()` 要的是 `const char*`

## 【实战2B】处理游戏协议：解析二进制数据包

**游戏里的数据包不是文本，是二进制。** 比如玩家的移动协议：

```
| 协议号 (2字节) | 玩家ID (4字节) | X坐标 (4字节) | Y坐标 (4字节) |
|     0x01       |   12345        |    100        |    200        |
```

**解析这样的数据包：**

```cpp
// 收到原始数据
char rawData[14];
recv(clientSocket, rawData, sizeof(rawData), 0);

// 解析协议
unsigned short protocolId;   // 协议号
int playerId;               // 玩家ID
float posX, posY;           // 坐标

// 从字节流中提取数据（网络字节序转本机字节序）
// 复制4字节到整数变量（小端序转大端序）
memcpy(&playerId, rawData + 2, 4);
memcpy(&posX, rawData + 6, 4);
memcpy(&posY, rawData + 10, 4);

// 字节序转换（Windows是小端序，网络是大端序）
playerId = ntohl(playerId);  // 网络字节序转本机字节序
posX = ntohl(*(int*)&posX);
posY = ntohl(*(int*)&posY);

cout << "玩家 " << playerId << " 移动到 (" << posX << ", " << posY << ")" << endl;
```

**等价的 Lua 解析（对比）：**
```lua
-- Lua 更简单，因为 table 可以直接按位截取
local rawData = socket.receive(clientSocket, 14)
local protocolId = string.unpack(">I2", rawData)  -- 大端序无符号16位
local playerId = string.unpack(">I4", rawData, 3)
local posX = string.unpack(">f", rawData, 7)
local posY = string.unpack(">f", rawData, 11)
```

**对比思考：**
- C++ 要手动处理字节序、内存对齐
- Lua 有 `struct` 库或 `string.unpack`，更方便
- 但 C++ 更底层，控制力更强

## 【实战2C】支持多客户端连接

**多客户端服务器需要用线程或者 select()。这里用最简单的方式——每来一个连接就开一个线程处理。**

```cpp
#include <iostream>
#include <winsock2.h>
#include <thread>       // C++11 线程
#include <vector>
#include <mutex>
#pragma comment(lib, "ws2_32.lib")
using namespace std;

vector<SOCKET> clients;  // 所有连接的客户端
mutex clientsMutex;      // 保护 clients 列表

// 处理单个客户端
void handleClient(SOCKET clientSocket) {
    cout << "[新线程] 开始处理客户端" << endl;

    while (true) {
        char buffer[1024] = {0};
        int recvLen = recv(clientSocket, buffer, sizeof(buffer), 0);

        if (recvLen <= 0) {
            cout << "[线程] 客户端断开" << endl;
            break;
        }

        buffer[recvLen] = '\0';
        cout << "[收到] " << buffer << endl;

        // 广播给所有客户端（群聊）
        lock_guard<mutex> lock(clientsMutex);
        for (SOCKET c : clients) {
            string msg = "[广播] " + string(buffer);
            send(c, msg.c_str(), msg.length(), 0);
        }
    }

    // 客户端断开，从列表移除
    lock_guard<mutex> lock(clientsMutex);
    for (auto it = clients.begin(); it != clients.end(); it++) {
        if (*it == clientSocket) {
            clients.erase(it);
            break;
        }
    }
    closesocket(clientSocket);
}

int main() {
    WSADATA wsa;
    WSAStartup(MAKEWORD(2, 2), &wsa);

    SOCKET listenSocket = socket(AF_INET, SOCK_STREAM, 0);
    sockaddr_in serverAddr;
    serverAddr.sin_family = AF_INET;
    serverAddr.sin_port = htons(8888);
    serverAddr.sin_addr.s_addr = INADDR_ANY;

    bind(listenSocket, (sockaddr*)&serverAddr, sizeof(serverAddr));
    listen(listenSocket, 10);
    cout << "[服务器] 监听端口 8888，支持多客户端..." << endl;

    while (true) {
        sockaddr_in clientAddr;
        int clientLen = sizeof(clientAddr);
        SOCKET clientSocket = accept(listenSocket, (sockaddr*)&clientAddr, &clientLen);

        {
            lock_guard<mutex> lock(clientsMutex);
            clients.push_back(clientSocket);
        }

        cout << "[服务器] 客户端已连接，当前在线: " << clients.size() << endl;

        // 为每个客户端启动一个新线程
        thread t(handleClient, clientSocket);
        t.detach();  // 主线程不等待，交给后台运行
    }

    closesocket(listenSocket);
    WSACleanup();
    return 0;
}
```

**关键点：**
- `thread` 是 C++11 的多线程支持
- `mutex` 保证多线程同时访问 `clients` 列表时不出错
- `lock_guard` 是 RAII 风格的锁，自动释放
- `t.detach()` 让线程在后台运行，主线程继续接受新连接

## 本周总结

| 概念 | 记住一句话 |
|------|-----------|
| if / else | 条件加括号，代码块加 `{}` |
| `=` vs `==` | `=` 赋值，`==` 比较 |
| for vs while | 知道次数用 for，不知道用 while |
| recv 返回值 | `> 0` 有数据，`= 0` 断开，`< 0` 错误 |
| 线程 | 每个客户端一个线程，`thread` + `mutex` |
