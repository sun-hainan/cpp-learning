## 结构体：自定义数据类型

### 为什么需要结构体？

在 Lua 里，你想描述一个玩家：
```lua
player = {
    name = "战士",
    level = 30,
    hp = 100,
    mp = 50
}
print(player.name)  -- "战士"
```

在 C++ 里，用**结构体**实现同样的功能：

```cpp
struct Player {
    string name;
    int level;
    int hp;
    int mp;
};

Player player;
player.name = "战士";
player.level = 30;
player.hp = 100;
player.mp = 50;

cout << player.name;  // "战士"
```

### 结构体的声明和初始化

**方法1：先定义再赋值**
```cpp
struct Player {
    string name;
    int level;
    int hp;
};

Player p1;
p1.name = "战士";
p1.level = 30;
p1.hp = 100;
```

**方法2：定义时直接初始化**
```cpp
Player p2 = {"法师", 25, 80};
```

**方法3：指定字段名初始化（C++11）**
```cpp
Player p3 = {.name = "道士", .level = 20, .hp = 60};
```

### 结构体数组

**存多个玩家：**
```cpp
Player party[3] = {
    {"战士", 30, 100},
    {"法师", 25, 80},
    {"道士", 20, 60}
};

for (int i = 0; i < 3; i++) {
    cout << party[i].name << " 等级" << party[i].level << endl;
}
```

### 结构体和函数

**传递结构体给函数：**
```cpp
void printPlayer(Player p) {  // 值传递，复制一份
    cout << p.name << " 等级" << p.level << " HP" << p.hp << endl;
}
```

**用引用避免复制（更高效）：**
```cpp
void levelUp(Player& p) {  // 引用，不复制，直接改原数据
    p.level++;
    p.hp += 10;
}
```

### 练习：技能结构体

**题目：** 定义技能结构体，包含名称、冷却时间、伤害值。创建3个技能输出信息。

```cpp
#include <iostream>
#include <string>
using namespace std;

struct Skill {
    string name;   // 技能名称
    int cooldown;   // 冷却时间（秒）
    int damage;     // 伤害值
};

int main() {
    Skill skills[3] = {
        {"火球术", 5, 100},
        {"冰冻术", 8, 80},
        {"雷电术", 10, 150}
    };

    cout << "===== 技能列表 =====" << endl;
    for (int i = 0; i < 3; i++) {
        cout << i+1 << ". " << skills[i].name
             << " 冷却:" << skills[i].cooldown << "秒"
             << " 伤害:" << skills[i].damage << endl;
    }

    return 0;
}
```

## 动态内存：new 和 delete

### 为什么需要动态内存？

**普通变量**（在栈上分配，函数结束后自动释放）：
```cpp
void createPlayer() {
    Player p;        // 栈上分配，函数结束时自动销毁
    p.level = 10;
}  // p 在这里消失，里面的数据也没了
```

**动态分配**（在堆上分配，手动管理）：
```cpp
void createPlayer() {
    Player* p = new Player();  // 堆上分配，永远存在
    p->level = 10;

    delete p;  // 手动释放内存！如果忘了，就会内存泄漏
}
```

### new 和 delete 配对使用

```cpp
// 分配单个变量
int* p = new int;     // 分配一个int的空间
*p = 100;
delete p;              // 释放

// 分配数组
int* arr = new int[5];  // 分配5个int的空间
arr[0] = 1;
arr[1] = 2;
delete[] arr;           // 释放数组（注意[]）

// 分配结构体
Player* p = new Player();
p->name = "战士";
p->level = 10;
delete p;
```

**指针访问结构体成员用 `->` 而不是 `.`：**
```cpp
Player* p = new Player();
p->name = "法师";    // 等价于 (*p).name = "法师"
p->level = 25;
delete p;
```

### 内存泄漏：忘了 delete

```cpp
void leak() {
    while (true) {
        int* p = new int[1000];  // 每次循环分配1KB
        // 忘了 delete p
    }
}  // 循环1万次 = 泄漏10MB（程序结束才释放）
```

**长期运行的程序（如游戏服务器）必须小心内存泄漏，否则内存会被慢慢耗光。**

### Lua 的 GC vs C++ 的手动管理

| 对比 | Lua | C++ |
|------|-----|-----|
| 内存分配 | `table.insert()` 自动 | `new` 手动申请 |
| 内存释放 | GC 自动回收 | `delete` 手动释放 |
| 优点 | 不会忘记，不会泄漏 | 精确控制，性能好 |
| 缺点 | GC 有开销，不知道何时回收 | 忘了 delete 会泄漏 |

## vector：动态数组

### 为什么需要 vector？

普通数组的大小是固定的：
```cpp
int arr[5];  // 只能存5个，不能多不能少
```

vector 的大小可以动态变化：
```cpp
#include <vector>
vector<int> v;   // 空vector
v.push_back(1);  // 动态增长
v.push_back(2);
v.push_back(3);
cout << v.size();  // 3
```

**和 Lua table 的对比：**

```lua
v = {}           -- 空table
table.insert(v, 1)  -- 追加
table.insert(v, 2)
print(#v)           -- 长度：2
```

```cpp
vector<int> v;   // 空vector
v.push_back(1);  // 追加
v.push_back(2);
cout << v.size();   // 2
```

### vector 常用操作

```cpp
#include <vector>

vector<int> v;           // 定义空vector
vector<int> v2(5);       // 定义有5个元素（默认0）的vector
vector<int> v3(5, 100);  // 定义有5个元素，每个都是100

v.push_back(10);         // 末尾添加
v.pop_back();            // 末尾删除
v.size();                // 元素个数
v.empty();               // 是否为空
v.clear();               // 清空所有元素
v[0];                    // 访问（不检查越界）
v.at(0);                 // 访问（会检查越界，越界抛异常）
```

### 遍历 vector

```cpp
vector<int> v = {10, 20, 30, 40, 50};

// 方法1：普通for
for (int i = 0; i < v.size(); i++) {
    cout << v[i] << " ";
}

// 方法2：范围for（最简洁）
for (int x : v) {
    cout << x << " ";
}

// 方法3：迭代器
for (auto it = v.begin(); it != v.end(); it++) {
    cout << *it << " ";
}
```

### 练习：用 vector 实现动态数组

```cpp
#include <iostream>
#include <vector>
using namespace std;

int main() {
    vector<int> v;
    int x;

    cout << "输入整数（输入0结束）:" << endl;
    while (cin >> x && x != 0) {
        v.push_back(x);
    }

    cout << "你输入了 " << v.size() << " 个数" << endl;
    cout << "分别是: ";
    for (int x : v) cout << x << " ";
    cout << endl;

    return 0;
}
```

## map：键值对

### map 是什么？

map 是"键值对"的容器，给你一把钥匙，存一个值进去：
```cpp
#include <map>

map<string, int> playerLevel;  // 键是string，值是int

playerLevel["战士"] = 30;
playerLevel["法师"] = 25;
playerLevel["道士"] = 20;

cout << playerLevel["战士"];  // 输出：30
```

**和 Lua table 的对比：**

```lua
playerLevel = {
    ["战士"] = 30,
    ["法师"] = 25,
    ["道士"] = 20
}
print(playerLevel["战士"])  -- 输出30
```

几乎完全一样！map 就是 C++ 里的"lua table（键值对模式）"。

### map 常用操作

```cpp
map<string, int> m;

// 插入
m["战士"] = 30;
m.insert({"法师", 25});       // 另一种插入方式
m.emplace("道士", 20);        // 更高效的插入（C++11）

// 查找
if (m.count("战士")) {        // 检查键是否存在
    cout << m["战士"];
}

auto it = m.find("法师");     // 返回迭代器
if (it != m.end()) {
    cout << it->second;       // 访问值
}

// 删除
m.erase("法师");

// 遍历
for (auto& kv : m) {
    cout << kv.first << " : " << kv.second << endl;
    // first = 键，second = 值
}
```

### 练习：学生成绩系统

```cpp
#include <iostream>
#include <map>
#include <string>
using namespace std;

int main() {
    map<string, int> scores;

    // 添加成绩
    scores["张三"] = 85;
    scores["李四"] = 92;
    scores["王五"] = 78;

    // 查找
    string name;
    cout << "输入要查找的学生姓名: ";
    cin >> name;

    if (scores.count(name)) {
        cout << name << "的成绩是: " << scores[name] << endl;
    } else {
        cout << "未找到该学生" << endl;
    }

    // 遍历所有
    cout << "\n===== 所有成绩 =====" << endl;
    for (auto& kv : scores) {
        cout << kv.first << " : " << kv.second << endl;
    }

    return 0;
}
```

## 本周总结

| 概念 | 记住一句话 |
|------|-----------|
| struct | 自定义数据类型，把相关字段组合在一起 |
| `->` | 指针访问结构体成员的运算符 |
| new/delete | 手动申请/释放堆内存，必须配对使用 |
| 内存泄漏 | 忘了 delete，内存永远不会被回收 |
| vector | 动态大小的数组，类似 Lua table 的数组模式 |
| map | 键值对，类似 Lua table 的键值模式 |
| auto | 自动推断类型，简化迭代器写法 |
