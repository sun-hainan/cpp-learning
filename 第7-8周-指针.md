# 第4周：指针——C++的灵魂

## 指针是什么？为什么 Lua 没有指针？

**先用一个生活例子：**

想象你家有一排储物柜，每个柜子有一个房间号（地址），里面放着东西（数据）。

```
柜子号（地址）    柜子里的东西（数据）
─────────────────────────────────────
0x0001            你的外套
0x0002            你的书
0x0003            你的水杯
```

**普通变量**告诉你"柜子里有什么"：
```cpp
int age = 28;
// 内存中：有一个柜子，柜子号是（比如）0x0010，柜子里放着28
cout << age;  // 看到柜子里的东西：28
```

**指针变量**告诉你"柜子的地址是什么"：
```cpp
int age = 28;       // 普通变量：柜子里放着28
int* p = &age;      // 指针变量：p存放着age这个柜子的地址
cout << p;          // 看到地址：0x0010
cout << *p;         // 通过地址找到柜子里的东西：28
```

**符号解释：**

| 符号 | 名称 | 含义 |
|------|------|------|
| `&` | 取地址运算符 | 拿到变量的"柜子号" |
| `*` | 解引用运算符 | 通过"柜子号"拿到柜子里的东西 |

```cpp
int age = 28;
int* p = &age;  // &age = 拿到age的地址，存到p里

*p = 30;        // 通过p里存的地址，找到那个柜子，把里面的东西改成30
cout << age;    // 输出：30（age被改了！）
```

## 为什么 Lua 没有指针？

因为 Lua 有 **GC（垃圾回收）**，它会自动整理内存，你不需要知道东西放在哪个柜子。

**C++ 需要指针的原因：**

1. **手动内存管理**：你想精确控制哪块内存用、哪块释放
2. **性能**：没有 GC 的开销，运行更快
3. **直接操作硬件**：游戏引擎需要直接控制内存来做优化

但代价是：你必须自己管理内存，用错了会崩溃（内存泄漏、野指针）。

## 指针的声明和使用

### 基本语法

```cpp
int age = 28;       // 普通变量
int* p;             // 声明一个"指向int"的指针变量（p存放地址）
p = &age;           // p存age的地址

cout << "地址:" << p << endl;    // 打印地址（十六进制）
cout << "值:" << *p << endl;     // 打印地址里的值：28
```

**注意：`int* p` 的写法，`*` 是类型的一部分，不是和 `p` 连在一起的。**

```cpp
int* p1;   // 声明指向int的指针
int *p2;   // 同上，写法不同，意义相同
int * p3;  // 同上
```

### 指针和数组的关系

**数组名就是指针！**

```cpp
int arr[5] = {10, 20, 30, 40, 50};

cout << arr << endl;      // 打印数组首元素的地址
cout << &arr[0] << endl;  // 打印第1个元素的地址（和上面一样！）

cout << *arr << endl;     // arr[0]的值：10
cout << *(arr+1) << endl; // arr[1]的值：20
```

**指针运算：**

```cpp
int arr[5] = {10, 20, 30, 40, 50};
int* p = arr;   // p指向arr[0]

p++;             // p指向下一个元素
cout << *p;      // 输出20（arr[1]）

p = p + 2;       // p再向后跳2个
cout << *p;      // 输出40（arr[3]）
```

```
arr   → 0x1000 → arr[0] = 10
arr+1 → 0x1004 → arr[1] = 20  （int占4字节，地址+4）
arr+2 → 0x1008 → arr[2] = 30
```

## 引用：C++的另一个利器

**引用是变量的别名（alias）：**

```cpp
int age = 28;
int& r = age;  // r 是 age 的引用

r = 30;         // 修改r，age也跟着变了
cout << age;    // 输出：30
```

**对比指针：**
```cpp
int age = 28;
int* p = &age;  // 指针存地址

*p = 30;        // 通过指针修改，要加*
cout << age;    // 输出：30
```

| 对比 | 指针 | 引用 |
|------|------|------|
| 初始化 | 可以为空 `int* p = nullptr` | 必须绑定一个变量 |
| 语法 | 用 `*` 解引用 | 直接当变量用 |
| 可变性 | 可以指向别的地址 `p = &other` | 绑定后不能改变 |
| 读法 | "p指向的int" | "r是age的引用" |

**什么时候用指针，什么时候用引用？**

| 场景 | 用什么 |
|------|--------|
| 函数参数需要"双向传递"（改外面的变量） | 引用 `void swap(int& a, int& b)` |
| 可能传空（不传任何东西） | 指针 `void setName(int* p)` |
| 不需要修改，只读取 | 普通值传递 `void print(int val)` |

## 指针在函数参数中的作用

**Lua 里函数参数是"按值传递"的：**
```lua
function change(x)
    x = 100  -- 改的是本地的x，不影响外面的变量
end

local a = 5
change(a)
print(a)  -- 输出5，外面的a没变
```

**C++ 里默认也是按值传递：**
```cpp
void change(int x) {
    x = 100;  // 改的是本地的x，不影响外面的变量
}

int a = 5;
change(a);
cout << a;  // 输出5，外面的a没变
```

**用指针实现"双向传递"：**
```cpp
void change(int* p) {
    *p = 100;  // 通过地址修改外面的变量
}

int a = 5;
change(&a);     // 传入a的地址
cout << a;      // 输出100，外面的a被改了！
```

**用引用实现"双向传递"（更简洁）：**
```cpp
void change(int& r) {
    r = 100;    // 直接修改，语法上像普通变量
}

int a = 5;
change(a);      // 传入a本身
cout << a;      // 输出100！
```

**这是 C++ 相对于 Lua 的巨大优势：Lua 函数只能返回1个值，C++ 可以通过指针/引用让函数"输出"多个值。**

## 练习：交换两个数的值

**不用指针/引用，函数改不了外面的变量：**
```cpp
void swap_wrong(int x, int y) {
    int temp = x;
    x = y;
    y = temp;
    // 改的是本地的x和y，外面的a、b不变
}

int a = 3, b = 5;
swap_wrong(a, b);
cout << a << "," << b;  // 输出：3,5（没变！）
```

**用指针实现：**
```cpp
void swap_pointer(int* p, int* q) {
    int temp = *p;
    *p = *q;
    *q = temp;
}

int a = 3, b = 5;
swap_pointer(&a, &b);
cout << a << "," << b;  // 输出：5,3（交换成功！）
```

**用引用实现（更直观）：**
```cpp
void swap_ref(int& x, int& y) {
    int temp = x;
    x = y;
    y = temp;
}

int a = 3, b = 5;
swap_ref(a, b);
cout << a << "," << b;  // 输出：5,3
```

## 常见的指针错误

### 错误1：空指针
```cpp
int* p = nullptr;  // nullptr = 空指针（不指向任何地址）
cout << *p;        // 错误！访问空指针会导致程序崩溃
```

### 错误2：未初始化的指针
```cpp
int* p;          // 没有初始化，p里是随机值（野指针）
*p = 100;        // 危险！可能写到随机内存，程序崩溃
```

### 错误3：指针越界
```cpp
int arr[5] = {1, 2, 3, 4, 5};
int* p = arr;
p = p + 10;      // arr只有5个元素，p+10越界了
cout << *p;       // 危险！访问越界内存
```

### 错误4：返回局部变量的指针
```cpp
int* bad_function() {
    int x = 100;  // x 是局部变量，函数结束后被销毁
    return &x;     // 返回局部变量的地址
}                  // x 被销毁，指针指向的内存已经无效

int* p = bad_function();
cout << *p;        // 危险！x已经被销毁（栈帧回收）
```

## 本周总结

| 概念 | 记住一句话 |
|------|-----------|
| 指针 | 存放地址的变量，通过 `*` 解引用拿到值 |
| `&` | 取地址运算符，拿到变量的"柜子号" |
| `*` | 解引用运算符，通过柜子号拿到柜子里的东西 |
| 数组名 | 就是指针，指向首元素 |
| 引用 | 变量的别名，用起来像普通变量 |
| 值传递 vs 引用传递 | 值传递：副本，不改原值；引用传递：别名，改原值 |
