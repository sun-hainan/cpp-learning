## 1.1 C++ 是什么？为什么游戏公司用它？

**一个比喻：**

- **Lua 像自动档汽车**：挂挡就走，不用关心发动机原理
- **C++ 像手动档汽车**：每个操作都要你亲自控制，但马力大、速度快

你写传奇脚本用的是 Lua，引擎底层是 C++。游戏公司选择 C++ 的原因很简单：**快、可控、跨平台**。

| 对比 | Lua | C++ |
|------|-----|-----|
| 运行速度 | 较慢（虚拟机解释执行） | 极快（编译后直接跑） |
| 内存管理 | 自动GC | 手动管理（但也更精准） |
| 类型检查 | 动态（自动推断） | 静态（先声明再使用） |
| 学习门槛 | 低 | 较高 |
| 应用场景 | 热更新脚本、逻辑层 | 核心引擎、服务器 |

## 1.2 编译原理：你的代码是怎么变成程序的？

**Lua 的运行方式：**
```bash
lua hello.lua
# 解释器一行一行读，直接执行
```

**C++ 的运行方式：**
```bash
# 两步走：先编译，再运行
cl /EHsc hello.cpp /Fe:hello.exe   # 编译
hello.exe                            # 运行
```

**编译的三个阶段：**

```
源代码（.cpp）
    ↓ 预处理器（展开 #include）
预处理后代码
    ↓ 编译器（翻译成汇编）
汇编文件（.asm）
    ↓ 汇编器（翻译成机器码）
目标文件（.obj）
    ↓ 链接器（合并库、启动代码）
可执行文件（.exe）
```

**为什么分这么细？**
- 预处理器处理 `#include`（把头文件内容复制进来）
- 编译器把 C++ 代码转成汇编（平台相关）
- 链接器把所有 `.obj` 文件合并，加上操作系统的启动代码

## 1.3 Hello World 逐行解析

```cpp
#include <iostream>      // ① 引入输入输出工具箱
using namespace std;     // ② 使用标准命名空间

int main() {            // ③ 程序入口
    cout << "Hello, World!" << endl;  // ④ 打印内容
    return 0;            // ⑤ 正常退出
}
```

**逐行解释：**

```cpp
#include <iostream>
// iostream = Input Output Stream（输入输出流）
// 相当于 Lua 里的 require "io"
// #include 是预处理器指令，在编译前把 iostream 的内容复制进来

using namespace std;
// std 是标准命名空间，里面有 cin、cout、endl 等工具
// 有了这行，cout 就直接写，不用写 std::cout

int main() {
// int = 这个函数的返回值类型是整数
// main = 特殊函数名，程序从这里开始执行
// 括号 { 到 } 之间是函数体

    cout << "Hello, World!" << endl;
    // cout = console output（控制台输出）
    // <<  = "插入运算符"，把后面的内容送进屏幕
    // endl = end line（换行）

    return 0;
    // 返回0表示程序正常结束
    // C++ 里函数要有返回值，Lua 不用管这个
}
```

**对比 Lua：**
```lua
print("Hello, World!")
-- Lua 里一句话搞定，C++ 要写这么多
-- 但 C++ 的代码更明确，编译器能帮你检查错误
```

## 1.4 变量：给数据起个名字

**Lua 的写法：**
```lua
name = "张三"
age = 28
hp = 100.0
```

**C++ 的写法（必须声明类型）：**
```cpp
string name = "张三";   // 字符串
int age = 28;           // 整数
double hp = 100.0;      // 浮点数
bool isAlive = true;    // 布尔
```

**为什么 C++ 要声明类型？**

Lua 的变量可以随时改变类型：
```lua
x = 10       -- x 是数字
x = "hello"  -- x 变成字符串，完全合法
```

C++ 声明后类型固定：
```cpp
int x = 10;
x = "hello";  // 编译错误！不能把字符串赋给 int
```

**类型的作用：**
- 告诉计算机分配多少内存
- 编译时发现错误，而不是运行到一半崩溃
- 编译器可以优化，运行更快

## 1.5 C++ 常用数据类型速查

| 类型 | 含义 | 例子 | 占用 |
|------|------|------|------|
| `int` | 整数 | `28`, `-100` | 4字节 |
| `double` | 浮点数 | `172.5`, `3.14` | 8字节 |
| `float` | 单精度浮点 | `172.5f` | 4字节 |
| `char` | 单个字符 | `'A'`, `'好'` | 1字节 |
| `bool` | 布尔 | `true`, `false` | 1字节 |
| `string` | 字符串 | `"张三"` | 动态 |

**和 Lua 的对应：**

```
Lua                C++
─────────────────────────
nil                 无对应
number（整数）       int
number（小数）       double
string              string（需 #include <string>）
boolean             bool
table               vector + struct + map
```

## 【实战1-2B】写一个游戏服务器最小框架

**目标：** 用 C++ 写一个最简单的游戏服务器，监听 8888 端口，收到客户端连接后打印消息。

**这是你 Lua 脚本工作的基础**——游戏服务器底层都是这样工作的。

```cpp
// server.cpp
#include <iostream>
#include <winsock2.h>
#pragma comment(lib, "ws2_32.lib")
using namespace std;

int main() {
    // ========== 1. 初始化 Winsock ==========
    WSADATA wsa;
    WSAStartup(MAKEWORD(2, 2), &wsa);
    cout << "[服务器] Winsock 初始化完成" << endl;

    // ========== 2. 创建监听 socket ==========
    SOCKET listenSocket = socket(AF_INET, SOCK_STREAM, 0);
    if (listenSocket == INVALID_SOCKET) {
        cout << "[错误] socket 创建失败" << endl;
        return 1;
    }
    cout << "[服务器] socket 创建成功" << endl;

    // ========== 3. 绑定端口 ==========
    sockaddr_in serverAddr;
    serverAddr.sin_family = AF_INET;          // IPv4
    serverAddr.sin_port = htons(8888);        // 端口号（游戏常用 8888）
    serverAddr.sin_addr.s_addr = INADDR_ANY;  // 接受任何IP的连接

    if (bind(listenSocket, (sockaddr*)&serverAddr, sizeof(serverAddr)) == SOCKET_ERROR) {
        cout << "[错误] 端口绑定失败，端口可能被占用" << endl;
        return 1;
    }
    cout << "[服务器] 绑定端口 8888 成功" << endl;

    // ========== 4. 开始监听 ==========
    if (listen(listenSocket, 5) == SOCKET_ERROR) {
        cout << "[错误] 监听失败" << endl;
        return 1;
    }
    cout << "[服务器] 正在监听 8888 端口..." << endl;

    // ========== 5. 接受客户端连接（阻塞等待）==========
    cout << "[服务器] 等待客户端连接..." << endl;
    sockaddr_in clientAddr;
    int clientLen = sizeof(clientAddr);
    SOCKET clientSocket = accept(listenSocket, (sockaddr*)&clientAddr, &clientLen);

    if (clientSocket == INVALID_SOCKET) {
        cout << "[错误] 接受连接失败" << endl;
        return 1;
    }
    cout << "[服务器] 客户端已连接！" << endl;

    // ========== 6. 接收数据 ==========
    char buffer[1024] = {0};
    int recvLen = recv(clientSocket, buffer, sizeof(buffer), 0);
    if (recvLen > 0) {
        buffer[recvLen] = '\0';  // 确保字符串以 \0 结尾
        cout << "[收到客户端消息] " << buffer << endl;
    }

    // ========== 7. 回复客户端 ==========
    const char* reply = "欢迎连接游戏服务器！";
    send(clientSocket, reply, strlen(reply), 0);
    cout << "[服务器] 已回复客户端" << endl;

    // ========== 8. 清理 ==========
    closesocket(clientSocket);
    closesocket(listenSocket);
    WSACleanup();
    cout << "[服务器] 已关闭" << endl;

    return 0;
}
```

**编译运行：**
```powershell
# 编译
cl /EHsc /std:c++17 server.cpp /Fe:server.exe /Fe:server.exe ws2_32.lib

# 运行（另一个 PowerShell 窗口）
.\server.exe
```

**测试：用 Lua 做客户端连接测试：**
```lua
-- 安装 socket：luarocks install luasocket
local socket = require("socket")
local client = socket.connect("127.0.0.1", 8888)
client:send("Hello Server!\n")
local response = client:receive()
print("服务器回复:", response)
client:close()
```

**服务器输出：**
```
[服务器] Winsock 初始化完成
[服务器] socket 创建成功
[服务器] 绑定端口 8888 成功
[服务器] 正在监听 8888 端口...
[服务器] 等待客户端连接...
[服务器] 客户端已连接！
[收到客户端消息] Hello Server!
[服务器] 已回复客户端
[服务器] 已关闭
```

**这个框架和你写传奇脚本的关系：**
- 你写的 Lua 逻辑层，底层就是通过这样的 socket 和客户端通信的
- 学完这个，你理解了你 Lua 脚本的"底层"是怎么工作的
